## 二、学习路线通用结构

每个学习路线包含：

1. **一句话总结** - 快速理解路线目标
2. **学习阶段总览** - 6个阶段的宏观规划
3. **阶段详情**（每个阶段包含）：
   - 目标描述
   - 知识点表格（模块、内容、预计时间）
   - 验证标准（可检查的里程碑）
   - 推荐资源（书籍、文档、课程）
4. **学习资源汇总** - 按阶段整理的书籍和平台
5. **关联索引** - 双向链接到其他相关文件
6. **下一步建议** -  actionable 的后续行动

## 四、使用建议

### 对于初学者
1. 选择1个方向（如Java后端或React前端）
2. 从阶段一开始，按顺序学习
3. 每个阶段完成后完成验证标准中的项目
4. 建立学习笔记，记录问题和解决方案

### 对于有经验的开发者
1. 根据当前水平选择起始阶段
2. 重点学习高级阶段（性能优化、架构设计）
3. 横向扩展，学习相关技术栈
4. 关注新兴技术部分，保持技术敏锐度

### 对于团队Leader
1. 使用学习路线作为团队培训大纲
2. 根据验证标准评估团队成员水平
3. 建立技术雷达，跟踪团队技术栈覆盖

## 五、Obsidian 使用指南

### 快速开始
1. 打开 `99-MOC索引/B-全栈技术学习路线索引.md`
2. 使用 Graph View 查看知识关联图谱
3. 点击任意链接跳转到技术栈或学习路线

### 日常学习
1. 在学习路线中找到当前阶段
2. 阅读知识点，完成验证标准
3. 在 `daily/` 目录创建学习笔记
4. 用 `#tag` 标记学习进度

### 知识管理
1. 定期回顾技术栈，更新热度评级
2. 补充新出现的技术到技术栈
3. 在学习路线中添加新的验证项目
